# Security Policy

## Reporting a Vulnerability (Private)
Please do not report security vulnerabilities in public issues.

Use GitHub's private vulnerability reporting flow:
- Go to: https://github.com/NorskNoobing/Unshittified-YouTube/security/advisories/new
- Fill in details, repro steps, impact, and affected version(s).

If the advisory link is unavailable, open an issue that says only "Security report requested" (no technical details), and we will move to a private channel.

## What to Include
- Clear reproduction steps
- Expected vs actual behavior
- Potential impact
- Browser and extension version
- Proof-of-concept (if available)

## Response Expectations
- Initial acknowledgement: within 7 days
- Status update: within 14 days
- Fix timeline: best effort based on severity and complexity

## Scope
This project is an unpacked/development-stage browser extension. Reports are still welcome for:
- Data exposure
- Privilege misuse
- Injection/XSS-style issues in extension pages
- Permission and origin handling flaws

## Safe Harbor
If you act in good faith, avoid privacy violations, and do not disrupt service, your research is welcome and appreciated.
